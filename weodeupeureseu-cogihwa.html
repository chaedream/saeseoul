워드프레스 초기화

워드프레스는 약간의 연습이 필요한 그런 툴이라는 생각을 종종 합니다. 웹 호스팅을 해도 처음에 시범 운영하는 기간을 가진 다음에 초기화 한 후 다시 사용하는것을 개인적으로 권합니다.

워드프레스 사이트나 블로그 등을 오랜기간 운영하신 분들이라도 종종 다시 시작할 때.. 깔끔하게 삭제를 하고 시작해야 하는 경우등이 있을 수 있겠죠..

이런 경우.. 누구나 깔끔하게 워드프레스를 삭제한 다음에 시작하시고 싶으실 겁니다.

워드프레스 초기화 방법은 여러가지가 있을 수 있지만, 크게 세가지 형태로 나누어 접근하시면 되겠습니다. 자신의 워드프레스 초기화 목적에 맞는 방법을 사용하시면 되겠습니다.

다만, 알아두실 부분은, 삭제는 '하드디스크'의 파일들을 삭제하는 것과 '데이타베이스'를 삭제하는 것.. 두가지를 해야 한다는 점은 숙지해 두실 필요가 있다 하겠습니다.

[방법.1 플러그인 활용하기]

워드프레스에는 별별 플러그인이 다 있죠.. 심지어 삭제하는 플러그인도 있습니다. -_- 여러가지 플러그인이 존재 하지만 개인적으로 WordPress Reset 플러그인을 권합니다.

설치 후..

1. 도구 > Reset.

2. Reset 입력 후 버튼을 클릭하면 초기화 됩니다.

이 방법으로 워드프레스 초기화를 할 때의 장점은 워드프레스 엔진 자체는 유지가 되기 때문에 다시 번거롭게 워드프레스 엔진을 다시 설치해야 하는 등의 과정이 필요 없다는 겁니다. 따라서, 워드프레스를 계속 사용하되 기타 다른 기능들을 초기화 하고 싶을 때 사용하시면 유용합니다.

다만, 업로드한 이미지도 남아있기는 하지만 데이타베이스에 저장된 이미지 정보는 모두 삭제되기 때문에 WP-content uploads 디렉터리에 저장된 이미지들을 모두 수동으로 삭제하시는 것을 권합니다.

[방법2. 웹 호스팅 회사 홈페이지 활용하기.]

다음으로, 웹 호스팅 회사의 홈페이지나 그들이 제공하는 기능을 활용하는 방법이 있습니다. 이는 각 웹 호스팅 회사들마다 제공하는 기능이나 방법들이 서로 다르겠죠?

이 방법은 엔진을 포함해 DB와 하드디스크의 모든 것이 초기화 되는 것입니다. 완전 삭제 후 워드프레스의 설치에서 부터 모두 새로 시작하시려고 하실 때 활용하시면 되겠습니다.

[방법3. FTP로 지우기.]

마지막으로, FTP로 인위적으로 삭제하는 방법이 있을 수 있습니다. 여기서 유의할 점은 최상위 폴더 자체를 삭제하시면 안된다는 겁니다. 제가 사용하는 웹호스팅의 경우 HTML 폴더 아래에 여러 폴더가 있는데요.. HTML 폴더 자체는 삭제하면 안되고 그 하위의 폴더를 삭제해야 합니다.

이 방법을 사용하실 때에는 DB 자체는 그대로 남아있다는 점에 유의해야 합니다. 따라서, DB의 데이터를 웹호스팅 회사에서 제공하는 기능으로 따로 삭제를 해 주셔야 합니다.

이런 방식은 하나의 웹호스팅 서버를 활용하면서 여러개의 워드프레스 사이트나 블로그 등을 운영하실 때 유용한 워드프레스 초기화 방법입니다.
