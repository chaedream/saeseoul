고정 IP 1개에 도메인 2개 연결하려고 할 때

web station 을 사용하여 홈페이지를 관리하고 있다면 가상 호스트 기능을 이용할 수 있으며, 혹은 역방향 프록시 설정으로 대응이 가능합니다. (이쪽은 80포트를 온전히 사용할 수 있어 좋습니다)

NAS1이 고정IP의 80포트를 점유하고 있기 때문에, NAS1의 설정에서 역방향 프록시를 설정합니다.

(DSM6 이전) 제어판 → 응용프로그램 포털 → "역방향 프록시" (DSM 7) 제어판 → 로그인 포털 → 고급 → "역방향 프록시"

프로토콜 : HTTP 혹은 HTTPS 중 선택 호스트 이름 : 새로 만든 도메인 포트 : 80

대상 프로토콜 : 위와 동일하게 호스트 이름 : NAS2의 내부IP 포트 : 7070

그냥 웹스테이션 에서 가상 호스트 이용하면 되지않을까 싶습니다. 도메인은 등록된 업체에서 DNS/A레코드 등록하면 동일한 IP라고 해도 주소찾아가서 웹 열어줄거구 개인적으로 포트번호만을 지정할 수 있는 역방향 프록시 보단, 웹스테이션이 다양한 접근이 가능해서 좋은것 같더군요.
